Flare animation test

Scratch file, not linked from the site. Three ways to make the cyan/orange .flare cloud drift and morph over time, in increasing order of how much of the gradient itself actually changes shape (versus the box it's drawn in just moving around).

A. Transform + hue-rotate the existing shape

Pure CSS, zero new tokens, works everywhere .flare already works. The gradient recipe is untouched; the whole blob translates/scales and its hue creeps. Reads as "drifting," not "morphing" — the cloud's internal shape stays fixed.

B. Animate the gradient's own stops (@property)

Genuinely morphs: center point and the cyan/orange split move independently over 20s. Needs modern browser support for animatable custom properties (broadly available since 2024); older browsers just see the static gradient, no animation, no breakage.

C. JS rAF, gradient string rebuilt per frame

Same visual morph as B, works in every browser (no @property dependency), at the cost of a small per-frame JS write. Good fallback path if B's compatibility ever becomes a concern.
