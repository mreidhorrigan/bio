Accepted tags & expected format
Core question tags
[Question.]Begins a question. Text after it (up to the next tag) is the question stem.
[Answer.]Marks the correct option. [Correct.] works identically.
[Correct.]Synonym for [Answer.].
[Distractor.]Marks an incorrect option. Use one per wrong choice.
[Question.] What is the powerhouse of the cell? [Answer.] Mitochondria. [Distractor.] Nucleus. [Distractor.] Ribosome.
Layout tags
[Page break.]In its own paragraph: starts a new page. At the end of a question: leaves the rest of that page blank for writing.
[Paragraph.]Marks the paragraph as scenario/description text (not a question, no number). Use for shared scenarios.
[Keep with previous.]Used with [Paragraph.]: keeps this paragraph attached to the one before it for multi-paragraph scenarios.
[Not a question.]Older synonym for [Paragraph.], still accepted.
[Image: name.png]Centers an uploaded image at this point. Optional: images embedded directly in the Word document are detected automatically (even on their own line, with no tag) and need no [Image:] tag.
Version & pooling tags
[Only Version X.]Before a question: it appears only in version X (A–E).
[Each version take N of the following options.]Followed by [Option.] paragraphs: each version gets N of them, distinct across versions when enough exist.
[Scramble the order of the following options.]Followed by [Option.] paragraphs: every version includes all, in a different order.
[Option.]Prefixes a question that belongs to the pool directly above.
Options are lettered (a) (b) (c)… after scrambling. In the answer key the correct option is prefixed [Answer.]. The test form shows no marker. Underlined text and embedded images are preserved in both outputs, and every page is numbered "Page X of Y".
